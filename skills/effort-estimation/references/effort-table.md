# 研发工时基准表（人日 pd）

本表为"工作项类型 × 复杂度"的基准工时（单位：人日 pd），用于 `effort-estimation` skill 查表计算。用户可直接修改本表，无需改动 SKILL.md。

## 核心约定：每行的"工作内容"粒度

从 v3 起，输出工作详单新增 **`工作内容`** 列，用于记录该行具体做什么。不同工作项类型填法不同：

| 类型 | 行粒度            | `工作内容` 列填法                                                    |
| ---- | ----------------- | -------------------------------------------------------------------- |
| 需求 | 1 行 / 业务模块   | `-`（模块级，内容由子任务清单隐含）                                  |
| 设计 | 1 行 / 业务模块   | `-`（模块级）                                                        |
| 前端 | 1 行 / 页面       | 页面名（"下单页" / "订单列表页" / "个人中心"）                       |
| 后端 | 1 行 / 业务动作   | 业务语言（"新增订单" / "查询订单详情" / "订单状态校验"）**禁写 URL** |
| 测试 | 1 行 / 业务模块   | `-`（模块级）                                                        |
| 运维 | 1 行 / 运维工作项 | 运维工作项（"部署流水线" / "监控告警" / "日志系统"），至少 2 行      |

**子任务**：架构图 · 接口设计 · DB / 数据模型设计 · 设计评审

### 前端（按页面，1 行/页）

| 复杂度 | 基准 pd | 典型形态                       |
| ------ | ------- | ------------------------------ |
| 简单   | 0.5     | 静态展示页 / 纯列表只读        |
| 中等   | 1.0     | 含表单 / 校验 / 多状态的交互页 |
| 复杂   | 2.0     | 富交互 / 实时推送 / 复杂状态机 |

**子任务**：详细设计 · 页面实现 · 自测 · 前后端联调 · 单测

### 后端（按业务增删查改 / 业务规则，1 行/业务动作）

| 复杂度 | 基准 pd | 典型形态                                   |
| ------ | ------- | ------------------------------------------ |
| 简单   | 0.3     | 单表字段查 / 删；无业务规则                |
| 中等   | 0.6     | 联表查 / 含业务规则的创建或更新 / 简单算法 |
| 复杂   | 1.2     | 事务 / 外部集成 / 状态机 / 复杂算法        |

**子任务**：详细设计 · 技术实现 · 单测 · 自测 · 前后端联调 · (如有) 第三方联调

**`工作内容` 填写约束**：必须用 **业务语言**，如"新增订单 / 查询订单列表 / 修改订单状态 / 删除订单 / 库存扣减逻辑 / 优惠计算"。**禁止** 出现 HTTP 方法（GET/POST/PUT/DELETE/PATCH）或 URL 路径。理由：这张表要给产品 / 项目经理看，而非接口文档。

| ------ | ------- |
| 简单   | 0.5     |

**子任务**：测试用例设计 · 用例执行 · 回归 · 缺陷复现

### 运维（按工作项，至少 2 行）

| 复杂度 | 基准 pd（单行） |
| ------ | --------------- |
| 简单   | 0.5             |
| 中等   | 1.0             |
| 复杂   | 2.0             |

**硬约束**：任何非平凡项目必须列 **至少 2 项** 运维工作内容（**部署 + 监控** 是最低配）。常规项目列 3-5 项，典型清单：

| 运维工作内容        | 常见复杂度档 | 何时需要                  |
| ------------------- | ------------ | ------------------------- |
| 部署流水线 / CI-CD  | 简单 ~ 中等  | **必列**                  |
| 监控告警            | 简单 ~ 中等  | **必列**                  |
| 日志 / 链路追踪     | 简单 ~ 中等  | 任何生产系统建议列        |
| 应急预案 / 运维文档 | 简单         | 对外或关键业务建议列      |
| 性能压测            | 中等 ~ 复杂  | 有性能 SLA（TPS / RT）时  |
| 灾备 / 多活         | 复杂         | 有高可用 SLA（≥ 99.9%）时 |

> 反推：典型项目总工时 60 pd，运维按全生命周期 5-10% = 3-6 pd。3-5 项 × 平均 1.0 pd = 3-5 pd，与此区间对齐。

## 复杂度判定准则

**新粒度下，复杂度由"单行工作项"的特征决定**，不再按整个"功能点"评分：

| 维度        | 前端（页面）            | 后端（业务动作）            | 运维（工作项）           |
| ----------- | ----------------------- | --------------------------- | ------------------------ |
| 数据 / 实体 | 页面字段数 / 图表数     | 涉及的表数 / 关联实体数     | -                        |
| 交互 / 规则 | 表单规则 / 交互状态数   | 业务规则 / 状态机复杂度     | 告警规则 / 压测场景数    |
| 集成 / 依赖 | 是否调多个后端 / 第三方 | 是否对接外部系统 / 消息队列 | 是否多活 / 跨云 / 多机房 |

三档定性：

- **简单**：单一实体 / 无规则 / 无集成
- **中等**：2-3 实体联动 / 含规则 / 最多 1 个集成点
- **复杂**：多实体 / 事务 / 状态机 / 多集成

边界落判：取下档并在 `备注` 列标注 `⚠ 边界偏低`，不阻塞式询问。

## 团队构成调整因子

加权生产率系数 `k` 按团队角色分布计算：

```
k = (senior × 0.8 + middle × 1.0 + junior × 1.3) / total
```

最终工时 = 基准工时 × k（四舍五入到 0.5 pd 粒度，保留 1 位小数）

### 角色系数

| 角色   | 系数 | 说明                             |
| ------ | ---- | -------------------------------- |
| senior | 0.8  | 资深研发，产出效率高于基准       |
| middle | 1.0  | 中级研发，与基准一致             |
| junior | 1.3  | 初级研发，需更多时间完成同样工作 |

### 预设团队构成

| 预设   | senior | middle | junior | k    |
| ------ | ------ | ------ | ------ | ---- |
| 偏资深 | 2      | 1      | 0      | 0.87 |
| 均衡   | 1      | 2      | 1      | 1.03 |
| 偏初级 | 0      | 1      | 2      | 1.20 |

### 示例

后端-中等（基准 0.6 pd）的"新增订单"业务动作，团队 `senior=1,middle=2,junior=1`：

- k = (1×0.8 + 2×1.0 + 1×1.3) / 4 = 1.025
- 最终工时 = 0.6 × 1.025 = 0.615 → 就近 0.5 粒度 = **0.5 pd**

## 非功能要求处理规则

非功能要求（NFR：性能、可用性、合规等）**不另起第 7 类模块**，按以下两条路径映射：

| NFR 类型                | 处理方式                                                            |
| ----------------------- | ------------------------------------------------------------------- |
| 性能（TPS / RT / 并发） | 相关后端业务动作复杂度提升一档（中等 → 复杂）+ 运维列一项"性能压测" |
| 可用性 / 灾备           | 运维列一项"灾备 / 多活"（复杂 2.0 pd）                              |
| 数据保留 / 审计 / 合规  | 后端新增"归档 / 冷热分层" 中等 1 行 + 运维列"数据保留策略"          |
| 安全（加密、脱敏）      | 相关后端动作升档 + 运维列"密钥管理 / 安全审计"                      |

## 人工复核触发条件

- 单工作项工时 > 5 pd → 提示拆细
- 单业务模块合计 > 25 pd → 建议二次拆分
- 复杂度判定落在边界 → 备注列标注 `⚠ 边界偏低`
- 运维行数 < 2 → **强制补齐**（至少 部署 + 监控）
